# Запуск Telegram-бота на Railway

## 1. Что загрузить в GitHub
Загрузите в репозиторий все файлы из этого архива:

- `bot.py`
- `config.py`
- `database.py`
- `tasks.py`
- `utils.py`
- `excel_utils.py`
- `docx_utils.py`
- `reminders.py`
- `requirements.txt`
- `Procfile`
- `.gitignore`

## 2. Railway Variables
В Railway откройте проект → ваш сервис → **Variables** и добавьте:

```text
TELEGRAM_TOKEN=токен_бота_из_BotFather
DEEPSEEK_API_KEY=ключ_DeepSeek
OPENAI_API_KEY=ключ_OpenAI
ADMIN_IDS=ваш_telegram_id
DB_FILE=/data/bot_data.db
VISION_MODEL=gpt-4o-mini
# Необязательно, но удобно для кэша файлов:
EXCEL_STORAGE_DIR=/data/excel_cache
WORD_STORAGE_DIR=/data/word_cache
```

Если администраторов несколько:

```text
ADMIN_IDS=123456789,987654321
```

## 3. Railway Volume
Чтобы бот не забывал пользователей, задачи и историю после redeploy:

1. Откройте сервис бота в Railway.
2. Найдите раздел **Volumes**.
3. Нажмите **Add Volume**.
4. В поле **Mount path** укажите:

```text
/data
```

5. В Variables оставьте:

```text
DB_FILE=/data/bot_data.db
```

Это не папка на вашем компьютере. Это постоянная папка внутри Railway.

## 4. Проверка после деплоя
После запуска напишите боту в Telegram:

```text
/start
```

Затем проверьте команды:

```text
/help
/get_contact_info
/new_task
/my_tasks
```

## Напоминания

Теперь бот умеет ставить простые напоминания без отдельной команды. Примеры сообщений:

```text
Напомни мне в 15:00 что у меня совещание в 15:05
Напомни через 10 минут что проверить почту
Напомни 25.12.2026 в 18:00 что сдать отчет
```

Проверить активные напоминания можно командой:

```text
/my_reminders
```

Важно: напоминания хранятся в SQLite. Чтобы они не пропадали после redeploy, нужен Railway Volume и переменная:

```text
DB_FILE=/data/bot_data.db
```


## Анализ изображений

Теперь бот умеет отвечать по картинкам. Примеры:

```text
[отправьте фото с подписью] Что изображено на этой фотографии?
[отправьте скриншот с подписью] Почему здесь ошибка?
[отправьте фото документа] Прочитай основной текст и кратко перескажи.
```

Для анализа изображений используется OpenAI vision-модель. По умолчанию:

```text
VISION_MODEL=gpt-4o-mini
```

Если нужно качество выше и вас устраивает большая стоимость, в Railway Variables можно заменить на:

```text
VISION_MODEL=gpt-4o
```

Важно: DeepSeek в этой версии остаётся текстовым помощником. Картинки анализируются через OpenAI API по вашему `OPENAI_API_KEY`.

## Excel-файлы

Теперь бот умеет работать с `.xlsx`:

```text
[отправьте Excel-файл без подписи]
```

Бот прочитает файл, запомнит его для текущего чата и подскажет, какие вопросы можно задать. После этого можно написать отдельным следующим сообщением:

```text
Добавь лист Итоги и посчитай количество строк по каждому ответственному
```

Бот должен прислать новую изменённую копию `.xlsx`, а не таблицу для ручного копирования.

```text
[отправьте Excel-файл с подписью] Проанализируй просроченные задачи
```

Бот ответит текстом по содержимому таблицы.

```text
[отправьте Excel-файл с подписью] Добавь столбец Комментарий и отметь строки, где прогресс меньше 50%, как «Отстаёт»
```

Бот создаст изменённую копию `.xlsx` и пришлёт её обратно.

Также можно создать Excel с нуля обычным сообщением:

```text
Создай Excel-файл для контроля задач: задача, ответственный, срок, статус, прогресс, комментарий
```

Ограничения:

- поддерживается только современный формат `.xlsx`;
- старые `.xls`, `.xlsm`, `.xlsb`, `.ods` нужно сначала сохранить как `.xlsx`;
- бот создаёт новую копию файла, исходник не перезаписывается;
- очень большие книги анализируются по первым строкам каждого листа, поэтому для сложных правок лучше писать конкретно: какой лист, какой столбец и что изменить.

Для Excel добавлена зависимость:

```text
openpyxl>=3.1.0
```


## Word-файлы

Теперь бот умеет работать с `.docx`:

```text
[отправьте Word-файл без подписи]
```

Бот прочитает файл, запомнит его для текущего чата и предложит написать следующее действие. Например отдельным сообщением:

```text
Добавь в договор раздел «Этапность оплат»: аванс, оплата после передачи промежуточного результата, финальный платеж после подписания акта
```

или сразу отправьте файл с подписью:

```text
[отправьте DOCX с подписью] Добавь этапность оплат и пришли исправленную редакцию
```

Бот создаст новую копию `.docx` и пришлёт её обратно. Исходный файл не перезаписывается.

Также можно создать Word-файл с нуля:

```text
Создай договор в Word на выполнение проектных работ с разделами: предмет, сроки, этапность оплат, ответственность сторон
```

Ограничения:

- поддерживается только современный формат `.docx`;
- старые `.doc`, `.rtf`, `.odt` нужно сначала сохранить как `.docx`;
- сканированные договоры внутри Word как картинки бот может не прочитать как текст;
- сложные правки договоров лучше формулировать конкретно: какой раздел/пункт изменить и что добавить;
- бот готовит рабочую редакцию документа, финальную юридическую проверку всё равно лучше делать человеком.

Для Word используется уже существующая зависимость:

```text
python-docx>=0.8.11
```

## Работа в рабочих групповых чатах

Бот можно добавить в рабочий чат Telegram. В этой версии в группах он не отвечает на каждое сообщение: бот обрабатывает сообщение только если его явно позвали.

Бот сработает, если пользователь ответил на сообщение бота через Reply или использовал команду:

```text
/ask@username_бота вопрос или просьба
```

Если в BotFather выключить Privacy Mode, бот также сможет получать обычные текстовые `@username_бота`-упоминания. Код всё равно игнорирует сообщения без обращения, но технически Telegram будет доставлять боту больше сообщений.

Примеры:

```text
/ask@mitra_assistant_bot кратко сформулируй итог обсуждения
```

```text
/ask@mitra_assistant_bot зафиксируй договорённость: Иван готовит КП до пятницы, Мария проверяет смету
```

```text
/ask@mitra_assistant_bot какие договорённости записаны?
```

Команды для группы:

```text
/agreements — показать последние договорённости текущего чата
/save_agreement текст — записать договорённость вручную
```

Рекомендуемый безопасный режим:

```text
/setprivacy
выбрать вашего бота
Enable
```

В этом режиме Telegram надёжно доставляет боту команды и ответы на сообщения бота. Поэтому для обычных вопросов в группе используйте `/ask@username_бота вопрос`.

Если принципиально нужно писать именно `@username_бота вопрос` без команды `/ask`, в BotFather придётся поставить Privacy Mode = Disable. Тогда Telegram будет доставлять боту всю переписку группы, а код бота будет сам отбрасывать всё, где нет обращения к нему.

Также не делайте бота администратором группы без необходимости: бот-администратор тоже может получать больше сообщений.

Дополнительные триггеры можно задать в Railway Variables, например:

```text
GROUP_TRIGGER_WORDS=настенька,#настенька
```

Тогда бот будет реагировать не только на `@username_бота`, но и на сообщения вроде:

```text
Настенька, запиши договорённость: ...
```

## Онлайн-поиск в интернете

На сайте DeepSeek есть переключатель Internet Search, но в обычном API это не включается одной галочкой. В этом архиве добавлен отдельный механизм онлайн-поиска для Telegram-бота.

### Рекомендуемый вариант: Tavily + DeepSeek

В этом режиме бот ищет свежие источники через Tavily, а итоговый ответ формирует через DeepSeek.

В Railway Variables добавьте:

```text
TAVILY_API_KEY=ваш_ключ_Tavily
ONLINE_SEARCH_ENABLED=true
```

DeepSeek при этом остаётся основным «мозгом»:

```text
DEEPSEEK_API_KEY=ваш_ключ_DeepSeek
```

### Запасной вариант: OpenAI web search

Если `TAVILY_API_KEY` не задан, бот попробует использовать OpenAI web search через уже существующий `OPENAI_API_KEY`.

Необязательная переменная:

```text
ONLINE_SEARCH_MODEL=gpt-5.5
```

Если модель недоступна на вашем аккаунте OpenAI, замените значение на доступную модель с поддержкой web search.

### Как пользоваться

Принудительный поиск:

```text
/search последние новости по рынку цемента в России
```

Или обычным сообщением:

```text
Проверь в интернете актуальные цены на арматуру А500С
```

```text
Найди онлайн последние изменения СП 63
```

```text
Какие сегодня новости по строительной отрасли?
```

В группе бот, как и раньше, сработает только если его упомянуть:

```text
@username_бота /search актуальные изменения законодательства по строительству
```

или:

```text
@username_бота проверь в интернете, вышла ли новая редакция СП 63
```

### Настройки DeepSeek reasoning

Обычные ответы по-прежнему идут через `DEEPSEEK_MODEL`. По умолчанию:

```text
DEEPSEEK_MODEL=deepseek-chat
```

Если на вашем аккаунте DeepSeek доступна reasoning-модель, можно включить рассуждения:

```text
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_THINKING=true
DEEPSEEK_REASONING_EFFORT=high
```

Если после этого DeepSeek начнёт возвращать ошибку в Railway logs, верните:

```text
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_THINKING=false
```

### Важное ограничение

Бот не читает интернет постоянно. Онлайн-поиск включается только для команды `/search` или когда в сообщении есть явные слова вроде «найди в интернете», «проверь онлайн», «актуальные данные», «новости», «сегодня», «цены», «курс».

## Отправка сообщений в рабочий групповой чат из лички

Чтобы бот мог по твоей команде из лички писать в рабочий чат, нужно один раз привязать этот чат.

1. Добавь бота в нужную группу Telegram.
2. В этой группе напиши команду:

```text
/bind_chat рабочий
```

Если privacy mode включён и команда не срабатывает, напиши так:

```text
/bind_chat@USERNAME_БОТА рабочий
```

3. В личке с ботом можно отправлять так:

```text
/send_to_chat рабочий Вадим, нужно успеть сделать РПЗ по СОШ к 11 числу
```

Или обычной фразой:

```text
Напиши Вадиму в рабочий чат, что он должен успеть сделать РПЗ по СОШ к 11 числу
```

Посмотреть привязанные чаты:

```text
/group_chats
```

Отправлять сообщения в рабочие чаты могут только пользователи из `ADMIN_IDS`, если эта переменная заполнена.

## Контроль поручений сотрудникам из лички

В этой версии бот умеет ставить поручения на контроль из личного чата руководителя и затем сам писать сотруднику в привязанный рабочий чат.

Перед использованием:

1. Добавьте бота в нужный рабочий групповой чат.
2. В этом чате выполните команду привязки, например:

```text
/bind_chat спартака
```

3. Если хотите, чтобы бот упоминал сотрудника через `@username`, привяжите имя сотрудника в личке с ботом:

```text
/bind_employee Дима @telegram_username_димы
```

После этого можно написать боту в личку:

```text
По объекту спартака необходимо, чтобы Дима подготовил РПЗ на прогресс к пн. Пожалуйста уточняй у него раз в день, как продвигаются дела по этому вопросу
```

Бот отправит первое сообщение в чат `спартака`, сохранит контроль в базу и затем будет раз в день уточнять статус. Если сотрудник ответит на сообщение бота словом «готово», «сделано», «выполнено» и т.п., контроль автоматически закроется.

Команды:

```text
/followups — список активных контролей
/stop_followup ID — остановить контроль вручную
/bind_employee Дима @username — привязать имя к username
/employees — список привязанных сотрудников
```

Важно: для сохранения контролей после redeploy должен быть подключён Railway Volume и переменная:

```text
DB_FILE=/data/bot_data.db
```

## Операционное ядро: проекты, задачи, подзадачи и командный контекст

В этой версии добавлено полноценное ядро операционного ассистента.

### Главная идея

Бот больше не хранит рабочую информацию только как переписку. Он переводит важные сообщения в структуру:

- проекты/объекты;
- операционные задачи;
- подзадачи;
- ответственные сотрудники;
- статусы и история обновлений;
- события командной памяти;
- ежедневный контроль задач.

### Команды

Создать объект/проект:

```text
/project спартака ЖК Спартака
/project сош СОШ на 825 мест
```

Посмотреть объекты:

```text
/projects
```

Создать операционную задачу:

```text
/op_task По объекту Спартака нужно, чтобы Дима подготовил РПЗ к пн. Уточняй каждый день
```

Посмотреть сводку задач:

```text
/tasks
/tasks по объекту Спартака
/tasks по Диме
```

Детали задачи:

```text
/op_task_info 12
```

Обновить статус:

```text
/op_update 12 70% сделал расчетную часть, осталось оформление
```

Добавить подзадачу:

```text
/subtask 12 Проверить оформление РПЗ
```

Ежедневная сводка:

```text
/daily_summary
```

### Естественный язык

Можно писать в личку:

```text
По объекту Спартака необходимо что бы Дима подготовил РПЗ на прогресс к пн. Пожалуйста уточняй у него раз в день, как продвигаются дела по этому вопросу
```

Бот создаст задачу, попытается определить объект, ответственного, срок, чат и включит ежедневный контроль.

Если в тексте есть РПЗ, бот автоматически создаст типовые подзадачи для РПЗ.

### Групповые чаты и командный контекст

В группах бот по-прежнему отвечает только когда к нему обращаются:

```text
/ask@username_бота что у нас по РПЗ по Спартака?
```

или при ответе на сообщение бота.

При этом бот ведет две памяти:

1. Личный диалоговый контекст внутри группы: `group:chat_id:user:user_id`.
2. Общий рабочий контекст группы: структурированные задачи, статусы, договоренности, адресованные боту сообщения.

Это сделано, чтобы бот:

- помнил, что обсуждалось с разными сотрудниками;
- не повторял второму сотруднику ответ, который был предназначен первому;
- мог подсветить общий контекст, если он релевантен.

### Контроль задач

Если задача создана с фразами:

```text
уточняй каждый день
контролируй
спрашивай раз в день
проверяй статус
```

бот будет сам писать в привязанный рабочий чат и спрашивать статус.

Сотрудник должен отвечать reply на сообщение бота. Тогда статус автоматически запишется в историю задачи.

Примеры ответов сотрудника:

```text
Сделал 70%, осталось оформить выводы
```

```text
Готово
```

```text
Есть блокер: жду исходные данные
```

### Важно

Для сохранения задач после redeploy Railway обязательно используйте Volume и переменную:

```text
DB_FILE=/data/bot_data.db
```

Для рабочих чатов заранее привяжите чаты:

```text
/bind_chat спартака
/bind_chat сош
/bind_chat рпз
```

Для сотрудников желательно привязать имена:

```text
/bind_employee Дима @dima_username
/bind_employee Егор @egor_username
```
