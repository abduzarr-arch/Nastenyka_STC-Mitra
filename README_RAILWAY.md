# Запуск Telegram-бота на Railway

## 1. Что загрузить в GitHub
Загрузите в репозиторий все файлы из этого архива:

- `bot.py`
- `config.py`
- `database.py`
- `tasks.py`
- `utils.py`
- `excel_utils.py`
- `reminders.py`
- `requirements.txt`
- `Procfile`
- `.gitignore`

## 2. Railway Variables
В Railway откройте проект → ваш сервис → **Variables** и добавьте:

```text
TELEGRAM_TOKEN=токен_бота_из_BotFather
DEEPSEEK_API_KEY=ключ_DeepSeek
OPENAI_API_KEY=ключ_OpenAI
ADMIN_IDS=ваш_telegram_id
DB_FILE=/data/bot_data.db
VISION_MODEL=gpt-4o-mini
# Необязательно, но удобно для Excel-кэша:
EXCEL_STORAGE_DIR=/data/excel_cache
```

Если администраторов несколько:

```text
ADMIN_IDS=123456789,987654321
```

## 3. Railway Volume
Чтобы бот не забывал пользователей, задачи и историю после redeploy:

1. Откройте сервис бота в Railway.
2. Найдите раздел **Volumes**.
3. Нажмите **Add Volume**.
4. В поле **Mount path** укажите:

```text
/data
```

5. В Variables оставьте:

```text
DB_FILE=/data/bot_data.db
```

Это не папка на вашем компьютере. Это постоянная папка внутри Railway.

## 4. Проверка после деплоя
После запуска напишите боту в Telegram:

```text
/start
```

Затем проверьте команды:

```text
/help
/get_contact_info
/new_task
/my_tasks
```

- исправлен сценарий «сначала отправил Excel, потом отдельным сообщением попросил изменить» — бот теперь запоминает последний Excel-файл и присылает изменённую копию.


## Напоминания

Теперь бот умеет ставить простые напоминания без отдельной команды. Примеры сообщений:

```text
Напомни мне в 15:00 что у меня совещание в 15:05
Напомни через 10 минут что проверить почту
Напомни 25.12.2026 в 18:00 что сдать отчет
```

Проверить активные напоминания можно командой:

```text
/my_reminders
```

Важно: напоминания хранятся в SQLite. Чтобы они не пропадали после redeploy, нужен Railway Volume и переменная:

```text
DB_FILE=/data/bot_data.db
```


## Анализ изображений

Теперь бот умеет отвечать по картинкам. Примеры:

```text
[отправьте фото с подписью] Что изображено на этой фотографии?
[отправьте скриншот с подписью] Почему здесь ошибка?
[отправьте фото документа] Прочитай основной текст и кратко перескажи.
```

Для анализа изображений используется OpenAI vision-модель. По умолчанию:

```text
VISION_MODEL=gpt-4o-mini
```

Если нужно качество выше и вас устраивает большая стоимость, в Railway Variables можно заменить на:

```text
VISION_MODEL=gpt-4o
```

Важно: DeepSeek в этой версии остаётся текстовым помощником. Картинки анализируются через OpenAI API по вашему `OPENAI_API_KEY`.

## Excel-файлы

Теперь бот умеет работать с `.xlsx`:

```text
[отправьте Excel-файл без подписи]
```

Бот прочитает файл, запомнит его для текущего чата и подскажет, какие вопросы можно задать. После этого можно написать отдельным следующим сообщением:

```text
Добавь лист Итоги и посчитай количество строк по каждому ответственному
```

Бот должен прислать новую изменённую копию `.xlsx`, а не таблицу для ручного копирования.

```text
[отправьте Excel-файл с подписью] Проанализируй просроченные задачи
```

Бот ответит текстом по содержимому таблицы.

```text
[отправьте Excel-файл с подписью] Добавь столбец Комментарий и отметь строки, где прогресс меньше 50%, как «Отстаёт»
```

Бот создаст изменённую копию `.xlsx` и пришлёт её обратно.

Также можно создать Excel с нуля обычным сообщением:

```text
Создай Excel-файл для контроля задач: задача, ответственный, срок, статус, прогресс, комментарий
```

Ограничения:

- поддерживается только современный формат `.xlsx`;
- старые `.xls`, `.xlsm`, `.xlsb`, `.ods` нужно сначала сохранить как `.xlsx`;
- бот создаёт новую копию файла, исходник не перезаписывается;
- очень большие книги анализируются по первым строкам каждого листа, поэтому для сложных правок лучше писать конкретно: какой лист, какой столбец и что изменить.

Для Excel добавлена зависимость:

```text
openpyxl>=3.1.0
```

## Работа в рабочих групповых чатах

Бот можно добавить в рабочий чат Telegram. В этой версии в группах он не отвечает на каждое сообщение: бот обрабатывает сообщение только если его явно позвали.

Бот сработает, если:

```text
@username_бота вопрос или просьба
```

или если пользователь ответил на сообщение бота через Reply.

Примеры:

```text
@mitra_assistant_bot кратко сформулируй итог обсуждения
```

```text
@mitra_assistant_bot зафиксируй договорённость: Иван готовит КП до пятницы, Мария проверяет смету
```

```text
@mitra_assistant_bot какие договорённости записаны?
```

Команды для группы:

```text
/agreements — показать последние договорённости текущего чата
/save_agreement текст — записать договорённость вручную
```

Чтобы Telegram сам не присылал боту всю переписку группы, настройте бота в BotFather:

```text
/setprivacy
выбрать вашего бота
Enable
```

Также не делайте бота администратором группы без необходимости. Если бот является администратором или Privacy Mode выключен, Telegram может доставлять ему все сообщения группы. Код всё равно будет игнорировать неадресованные сообщения, но для приватности лучше держать Privacy Mode включённым.

Дополнительные триггеры можно задать в Railway Variables, например:

```text
GROUP_TRIGGER_WORDS=настенька,#настенька
```

Тогда бот будет реагировать не только на `@username_бота`, но и на сообщения вроде:

```text
Настенька, запиши договорённость: ...
```

## Онлайн-поиск в интернете

На сайте DeepSeek есть переключатель Internet Search, но в обычном API это не включается одной галочкой. В этом архиве добавлен отдельный механизм онлайн-поиска для Telegram-бота.

### Рекомендуемый вариант: Tavily + DeepSeek

В этом режиме бот ищет свежие источники через Tavily, а итоговый ответ формирует через DeepSeek.

В Railway Variables добавьте:

```text
TAVILY_API_KEY=ваш_ключ_Tavily
ONLINE_SEARCH_ENABLED=true
```

DeepSeek при этом остаётся основным «мозгом»:

```text
DEEPSEEK_API_KEY=ваш_ключ_DeepSeek
```

### Запасной вариант: OpenAI web search

Если `TAVILY_API_KEY` не задан, бот попробует использовать OpenAI web search через уже существующий `OPENAI_API_KEY`.

Необязательная переменная:

```text
ONLINE_SEARCH_MODEL=gpt-5.5
```

Если модель недоступна на вашем аккаунте OpenAI, замените значение на доступную модель с поддержкой web search.

### Как пользоваться

Принудительный поиск:

```text
/search последние новости по рынку цемента в России
```

Или обычным сообщением:

```text
Проверь в интернете актуальные цены на арматуру А500С
```

```text
Найди онлайн последние изменения СП 63
```

```text
Какие сегодня новости по строительной отрасли?
```

В группе бот, как и раньше, сработает только если его упомянуть:

```text
@username_бота /search актуальные изменения законодательства по строительству
```

или:

```text
@username_бота проверь в интернете, вышла ли новая редакция СП 63
```

### Настройки DeepSeek reasoning

Обычные ответы по-прежнему идут через `DEEPSEEK_MODEL`. По умолчанию:

```text
DEEPSEEK_MODEL=deepseek-chat
```

Если на вашем аккаунте DeepSeek доступна reasoning-модель, можно включить рассуждения:

```text
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_THINKING=true
DEEPSEEK_REASONING_EFFORT=high
```

Если после этого DeepSeek начнёт возвращать ошибку в Railway logs, верните:

```text
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_THINKING=false
```

### Важное ограничение

Бот не читает интернет постоянно. Онлайн-поиск включается только для команды `/search` или когда в сообщении есть явные слова вроде «найди в интернете», «проверь онлайн», «актуальные данные», «новости», «сегодня», «цены», «курс».
